# @openmatch/matching

The open-source matching algorithm that powers OpenMatch's discovery deck. This package is the single source of truth — the backend imports it directly, the `/api/v1/transparency/algorithm/current` endpoint serves its `algorithm_config.json`, and the iOS app renders explanations generated by its `explain()` function.

## Quick start

```ts
import { getDiscoveryDeck, explain, currentConfig } from "@openmatch/matching";

const deck = getDiscoveryDeck({
  viewer,
  candidates,
  blocks,
  prevSwipes,
  now: new Date(),
  limit: 20,
});

// deck.cards.forEach(card => console.log(card.profileId, card.score, card.explanation));
```

## What's inside

- `src/types.ts` — plain TypeScript types. No coupling to Prisma or any backend.
- `src/eligibility.ts` — one predicate per rule in `/docs/algorithm/spec.md` §1.
- `src/scoring.ts` — pure scoring functions, one per component.
- `src/explain.ts` — plain-language explanations. Never quotes raw scores, never leaks private fields.
- `src/deck.ts` — `getDiscoveryDeck`, the deck-assembly entry point.
- `src/index.ts` — public API.
- `algorithm_config.json` — the live weights, constraints, and compatibility matrices.
- `fixtures/` — synthetic users and named scenarios for tests.
- `test/` — Vitest tests covering every rule in the spec.

## Auditing

```bash
npm install
npm test
```

Tests use only synthetic data from `fixtures/`. No production data is required, and the suite is deterministic.

## Changing the algorithm

See [`/docs/algorithm/versioning.md`](../docs/algorithm/versioning.md). Algorithm changes require a version bump, a changelog entry, fairness analysis, abuse analysis, and synthetic-fixture tests.
